# Contributing to Rook ♜

Thanks for wanting to help! Here's how to get started.

## 🚀 Quick Setup

```bash
# Fork & clone
git clone https://github.com/YOUR_USERNAME/rook.git
cd rook

# Install
npm install
cd frontend && npm install && cd ..

# Configure
cp .env.example .env
# Edit .env with your keys

# Run
npm run dev              # Backend :3001
npm run frontend:dev     # Frontend :3000
```

## 📋 What to Work On

Check the [Issues](https://github.com/rook/rook/issues) tab. Good first issues are labeled `good first issue`.

## 🎯 Guidelines

- **Keep it simple** — Rook's whole point is simplicity
- **One feature per PR** — Don't bundle unrelated changes
- **Test your changes** — Make sure it actually works
- **Follow existing code style** — Look at what's already there
- **Write good commit messages** — `feat: add X` or `fix: resolve Y`

## 🐛 Bug Reports

Open an issue with:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Screenshots if helpful

## 💡 Feature Requests

Open an issue with:
- What you want
- Why you want it
- How it fits Rook's mission (simplicity!)

## 🔒 Security

Found a security issue? Don't open a public issue. Email us instead.
